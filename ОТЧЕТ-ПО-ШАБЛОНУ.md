# ОТЧЁТ ПО ШАБЛОНУ

## Тема

**PPC Audit Workspace v.0** — веб-приложение для ускорения и стандартизации PPC-аудита рекламных кампаний Яндекс Директа.

## 1. Постановка задачи

### Проблема «до»

```mermaid
flowchart TD
    A[Маркетолог получает доступ к Директу] --> B[Скачивает Excel, скрины, заметки]
    B --> C[Вручную разбирает метрики и риски]
    C --> D[Пишет выводы и рекомендации в Word/Notion]
    D --> E[Собирает PDF для клиента]
    E --> F[3,5–5 часов на один отчёт]

    style F fill:#ffcdd2,stroke:#c62828
```

| Боль | Последствие |
|------|-------------|
| Разрозненные материалы | Потеря времени на сбор контекста |
| Нет единого шаблона выводов | Разное качество отчётов |
| AI «в чате» без контроля | Риск галлюцинаций в клиентском PDF |
| Нет журнала анализа | Сложно воспроизвести, что было отправлено в AI |

### Цель проекта

Сократить время подготовки PPC-аудита **на 35–45%**, сохранив **контроль маркетолога** над каждым выводом перед отправкой клиенту.

```mermaid

    subgraph Маркетолог
        M1[Загружает материалы]

    end

    subgraph Система
        S1[Парсит Excel / OCR]
        S2[Маскирует PII]

        S4[Quality guard]

    end

    subgraph AI
        A1[Черновик выводов]
        A2[Графики и КП]
    end

    M1 --> S1
    M2 --> S3
    M3 --> S3 --> A1
    A1 --> S4 --> M4
    M4 -->|подтверждено| M5 --> S5
    A2 --> S4

    M4 -.->|отклонено| M3
```

**Принцип:** AI — черновик; в PDF попадают только `human_confirmed` / `human_edited`.

## 4. Поток данных и AI-пайплайн

### От материалов до PDF

```mermaid
sequenceDiagram
    actor M as Маркетолог
    participant UI as Веб-панель
    participant API as FastAPI
    participant P as Privacy service
    participant MR as Model Router
    participant DB as SQLite

    M->>UI: Загрузить Excel, скрины
    M->>UI: Отметить «В AI»
    M->>UI: Запуск AI + согласие
    UI->>API: POST /analyze/start
    API->>P: Маскирование PII
    API->>MR: Промпт + контекст
    MR-->>API: JSON-ответ
    API->>API: Parse JSON + shape fill
    API->>API: Quality guard
    API->>DB: audit_runs.output_json
    API-->>UI: WebSocket 100%
    M->>UI: Confirm выводы
    M->>UI: Preview PDF
    UI->>API: GET /export/pdf
    API-->>M: PDF файл
```

### Цепочка надёжности AI

```mermaid
flowchart TD
    START[Запрос к Model Router] --> JSON1{JSON через<br/>unified ProxyAPI}
    JSON1 -->|OK| PARSE[parse_ai_response_json]
    JSON1 -->|fail| NATIVE[Native Anthropic]
    NATIVE --> PARSE

    PARSE -->|битый JSON| RETRY[Повтор с fix-промптом]
    RETRY --> PARSE

    PARSE -->|OK| SHAPE[ensure_ai_analysis_shape<br/>charts / schemes / КП]
    SHAPE --> VALID[Pydantic AIAnalysisResult]
    VALID --> GUARD[Quality guard<br/>needs_review]
    GUARD --> SAVE[Сохранение в БД]

    style SAVE fill:#c8e6c9,stroke:#2e7d32
```

### Модели в селекторе

| Модель | Назначение | Ориентир по стоимости |
|--------|------------|----------------------|
| Claude Opus 4.5 | Максимальное качество анализа | выше |
| Claude Sonnet 4.5 | Баланс цена/качество | средняя |
| Claude Haiku 4.5 | Быстрые черновики | ниже |
| GPT-4o | Fallback / альтернатива | средняя |
| GPT-4o mini | Оценка / экономичный прогон | низкая |
| Демо-режим | Сдача без ключей API | бесплатно |

Разработанное веб-приложение **PPC Audit Workspace v.0**:

- **ускоряет** подготовку клиентского PPC-отчёта;
- **стандартизирует** структуру выводов и PDF;
- **сохраняет ответственность** маркетолога через подтверждение каждого вывода;
- **воспроизводимо** запускается на чистой машине по инструкции из репозитория.

Проект готов к **демонстрации**, **защите** и **практическому использованию** в работе PPC-специалиста.
