## Вариант A — Docker (рекомендуется для проверяющего)

### Шаги

> **Важно:** не запускайте команды из `C:\WINDOWS\System32` — там нет прав на `git clone` (`Permission denied`) и нет `docker-compose.yml` (`no configuration file provided`). Сначала перейдите в папку пользователя или на диск `C:\`.

```powershell
cd $env:USERPROFILE\Documents
git clone https://github.com/Ekaterina-Kotendzhi/PPC-Audit-Workspace-v.0.git PPC-Audit-Workspace-v.0-test
cd PPC-Audit-Workspace-v.0-test
docker compose up --build
```

Альтернатива папки: `cd C:\PPC-Audit-Clean-test` (если создаёте вручную) или `cd C:\Users\ВашеИмя\Desktop`.

1. Дождаться `Application startup complete` (первый build: **5–15 мин**; нужен интернет — Playwright скачивает Chromium ~160 MB).
2. Открыть http://localhost:8000 — главная без ошибок.
3. **+ Новый аудит** → заполнить название клиента и нишу → сохранить.
4. Вкладка **Данные → Директ** → загрузить тестовый `.xlsx` Мастер-отчёта.
5. **Запуск AI** → принять согласие → дождаться 100% (демо-режим из `.env.docker`).
6. **Выводы** → подтвердить минимум **1** карточку.
7. **Отчёт** → открыть HTML/PDF preview (новая вкладка).
8. Открыть http://localhost:8000/audit-runs — есть запись `ai_analysis` со status `success`.

### Остановка

```powershell
docker compose down
```

### Ожидаемый результат

| Критерий | OK |
|----------|-----|
| UI открывается за < 30 сек после старта контейнера | ☐ |
| Excel загружается, на Директ видны KPI/риски | ☐ |
| AI завершается без 500 (демо) | ☐ |
| Findings появились на «Выводы» | ☐ |
| PDF/HTML preview открывается | ☐ |
| `audit_runs` содержит input_json и output_json | ☐ |

### Известные ограничения Docker

- **OCR скринов** отключён (`OCR_PROVIDER=disabled` в `.env.docker`) — для сдачи достаточно Excel.
- **Chroma/KB** использует локальные embeddings — без внешнего API.
- Данные сохраняются в `./data` на хосте (volume).

## Вариант B — Python venv (Windows)

```powershell
cd $env:USERPROFILE\Documents
git clone https://github.com/Ekaterina-Kotendzhi/PPC-Audit-Workspace-v.0.git PPC-Audit-Workspace-v.0-test
cd PPC-Audit-Workspace-v.0-test
powershell -ExecutionPolicy Bypass -File .\scripts\setup-venv.ps1
copy .env.example .env
# API-ключи НЕ заполнять
cd frontend
npm install
npm run build
cd ..
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Далее шаги 2–8 из варианта A (URL тот же).

**Время:** ~8–15 мин при первом `setup-venv` + `npm install`.

## Устранение проблем

| Симптом | Решение |
|---------|---------|
| `Permission denied` при `git clone` | Вы в `System32` или защищённой папке. Выполните `cd $env:USERPROFILE\Documents` и повторите clone |
| `no configuration file provided: not found` | Вы не в папке проекта. Сначала `cd ...\PPC-Audit-Workspace-v.0-test`, затем `docker compose up --build` |
| `cd PPC-Audit-Workspace-v.0-test` — путь не найден | Clone не выполнился из-за ошибки выше — исправьте каталог и снова `git clone` |
| Порт 8000 занят | `docker compose down`; или сменить порт в `docker-compose.yml` |
| Build падает на Playwright (timeout / EAI_AGAIN) | Стабильный интернет; повторить `docker compose build` (в Dockerfile таймаут 5 мин + retry) |
| Пустой UI (нет стилей/JS) | Пересобрать образ — frontend stage должен собрать `app.js` |
| AI failed | Убедиться, что `.env.docker` содержит `PPC_FORCE_DEMO_AI=true` |
| PDF пустой | Сначала confirm вывод на «Выводы», затем refresh summary на «Отчёт» |
