# Проверка на чистой машине (воспроизводимость)

Чеклист для сдачи блока **«Упаковка и воспроизводимость»**.  
Выполняется на ПК **без** предустановленного проекта (или в новой папке / VM / Docker с нуля).

**Репозиторий:** [github.com/Ekaterina-Kotendzhi/PPC-Audit-Workspace-v.0](https://github.com/Ekaterina-Kotendzhi/PPC-Audit-Workspace-v.0)  
**Версия:** последний commit в ветке `master` на GitHub (после `git clone` проверьте: `git log -1 --oneline`).

## Вариант A — Docker (рекомендуется для проверяющего)

### Шаги

> **Важно:** не запускайте команды из `C:\WINDOWS\System32` — там нет прав на `git clone` и нет `docker-compose.yml`. Сначала перейдите в папку пользователя.

```powershell
cd $env:USERPROFILE\Documents
git clone https://github.com/Ekaterina-Kotendzhi/PPC-Audit-Workspace-v.0.git PPC-Audit-Workspace-v.0-test
cd PPC-Audit-Workspace-v.0-test
git log -1 --oneline
docker compose up --build
```

1. Дождаться `Application startup complete` (первый build: **5–15 мин**).
2. Открыть http://localhost:8000 — главная без ошибок.
3. **+ Новый аудит** → название клиента, ниша, цель → сохранить.
4. **Данные → Директ** → загрузить тестовый `.xlsx` Мастер-отчёта.
5. (Опционально) **Источники** → отметить материалы **«Будет в AI»**.
6. **Запуск AI** → модалка: согласие → дождаться 100% (**демо** из `.env.docker`, ключи не нужны).
7. **Выводы** → подтвердить минимум **1** карточку.
8. **Отчёт** → открыть HTML/PDF preview (новая вкладка).
9. http://localhost:8000/audit-runs — запись `ai_analysis` со status `success`.

### Остановка

```powershell
docker compose down
```

### Ожидаемый результат

| Критерий | OK |
|----------|-----|
| UI открывается за < 30 сек после старта контейнера | ☐ |
| Excel загружается, на Директ видны KPI/риски | ☐ |
| AI завершается без 500 (демо) | ☐ |
| Findings появились на «Выводы» | ☐ |
| PDF/HTML preview открывается | ☐ |
| `audit_runs` содержит input_json и output_json | ☐ |

### Режимы AI в Docker

| Режим | Настройка | Для кого |
|-------|-----------|----------|
| **Демо (сдача)** | `.env.docker`: `PPC_FORCE_DEMO_AI=true`, ключи пустые | Проверяющий, офлайн-защита |
| **Внешний Claude** | `PPC_FORCE_DEMO_AI=false` + ключ ProxyAPI в `ANTHROPIC_API_KEY` | Разработка, боевая демонстрация |

> В репозитории по умолчанию `.env.docker` настроен на **демо без ключей**.

### Известные особенности Docker

- **OCR:** Tesseract установлен в образе; в `.env.docker` для сдачи часто `OCR_PROVIDER=disabled` — достаточно Excel. Для OCR: `OCR_PROVIDER=tesseract_cli`.
- **Chroma/KB:** локальные embeddings в демо — без внешнего API.
- Данные: volume `./data` на хосте.
- Логи: `LOG_LEVEL=DEBUG` в консоли (`docker logs -f ppc_audit_workspace`).

## Вариант B — Python venv (Windows)

```powershell
cd $env:USERPROFILE\Documents
git clone https://github.com/Ekaterina-Kotendzhi/PPC-Audit-Workspace-v.0.git PPC-Audit-Workspace-v.0-test
cd PPC-Audit-Workspace-v.0-test
powershell -ExecutionPolicy Bypass -File .\scripts\setup-venv.ps1
copy .env.example .env
# API-ключи НЕ заполнять — останется демо при пустых ключах
cd frontend
npm install
npm run build
cd ..
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --log-level debug --access-log
```

Далее шаги 2–9 из варианта A (URL тот же).

**Время:** ~8–15 мин при первом `setup-venv` + `npm install`.

## Устранение проблем

| Симптом | Решение |
|---------|---------|
| `Permission denied` при `git clone` | `cd $env:USERPROFILE\Documents`, повторить clone |
| `no configuration file provided` | `cd` в папку с `docker-compose.yml` |
| Порт 8000 занят | `docker compose down` или сменить порт в `docker-compose.yml` |
| Build падает на Playwright | Стабильный интернет; повторить `docker compose build` |
| Пустой UI (нет стилей/JS) | Пересобрать образ — frontend stage собирает `app.js` |
| AI failed (демо) | `.env.docker`: `PPC_FORCE_DEMO_AI=true`, ключи пустые |
| AI failed (Claude) | Проверить ключ ProxyAPI; логи: `docker logs ppc_audit_workspace`; commit ≥ `3551632` |
| `Validation error` charts/schemes | Обновить до commit `3551632`+ (автодополнение схемы) |
| PDF пустой | Подтвердить вывод на «Выводы», refresh summary на «Отчёт» |
| OCR «Tesseract не найден» | В Docker: образ уже с Tesseract; включить `OCR_PROVIDER=tesseract_cli` |
